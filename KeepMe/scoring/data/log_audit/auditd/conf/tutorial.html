Configuring auditd Settings

Referenced from BookofZeus and pages 195 and 197 of CIS Ubuntu Linux 16.04 Benchmark

Installing various packages can make your system vulnerable to any known or discovered security issues in the software. Hackers trying to gain access to your machine often can take advantage of such security holes in your system's software. For this reason, it is highly recommended to purge anything on your machine that is not required to be there.


Edit the auditd config file by typing sudo nano /etc/audit/auditd.conf and set the following parameters:

Parameter Name
Default
Secure
Details
max_log_file
6
6 or greater
The maximum log file storage should be large enough to retain and review any unauthorized activity on the system. The RHEL 6 STIG recommends this value to be 6MB or greater.
space_left_action
SYSLOG
email
This notifies the system administrator via email when the system's disk space is running low.
action_mail_acct
root
root
Specifies a target for sending notification emails. It requires having /usr/lib/sendmail on your machine.
admin_space_left_action
SUSPEND
halt
The system should halt when audit files have reached their maximum. While availability is a key element of security, in high security contexts it is more critical to ensure that all unauthorized activity is detected.
max_log_file_action
ROTATE
keep_logs
keep_logs prevents any old logs from getting deleted when the audit log file size reaches its maximum

Press Ctrl+O to save your changes. That should be it!
