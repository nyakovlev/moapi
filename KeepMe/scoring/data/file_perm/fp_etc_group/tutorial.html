Setting Correct File Permissions on /etc/group

Referenced from page 322 of CIS Ubuntu Linux 16.04 Benchmark

System files must have correct permissions to ensure that 1) various system utilities and applications can access them as needed, and 2) users or hackers cannot steal data or maliciously modify these files and make them behave unexpectedly.


/etc/group contains a list of all valid groups in the system. While it must be protected from unauthorized write access, it should remain readable, for it is needed in various programs.


  • The octal file permission of /etc/group must be set to 644 (-rw-r--r--).
  • The owner of /etc/group must be root (UID 0)
  • The group associated with /etc/group must be root (GID 0)


typing ls -l [filename] or stat [filename] will display the file's current permissions.


To change the file permission octal to 644, type sudo chmod 644 /etc/group.


To change the owner of the file to root, type sudo chown 0 /etc/group.


To change the file group to root, type sudo chgrp 0 /etc/group.


That should do it!
