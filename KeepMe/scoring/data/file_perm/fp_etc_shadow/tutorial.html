Setting Correct File Permissions on /etc/shadow

Referenced from page 321 of CIS Ubuntu Linux 16.04 Benchmark

System files must have correct permissions to ensure that 1) various system utilities and applications can access them as needed, and 2) users or hackers cannot steal data or maliciously modify these files and make them behave unexpectedly.


/etc/shadow stores critical security information about user accounts. If a hacker gains access to the file, they can crack password hashes stored in the file, and they can gain information about account status and expiration.


  • The octal file permission of /etc/shadow must be set to 640 (-rw-r-----).
  • The owner of /etc/shadow must be root (UID 0)
  • The group associated with /etc/shadow must be shadow (GID 42)


typing ls -l [filename] or stat [filename] will display the file's current permissions.


To change the file permission octal to 640, type sudo chmod 640 /etc/shadow.


To change the owner of the file to root, type sudo chown 0 /etc/shadow.


To change the file group to shadow, type sudo chgrp 42 /etc/shadow.


That should do it!
