Securing Default Password Expiration Values

Password expiration is configured by editing values in the /etc/login.defs file. Begin this by typing sudo nano /etc/login.defs.


Look for three lines in the file: PASS_MAX_DAYS, PASS_MIN_DAYS, and PASS_WARN_AGE. Lines starting with a pound (#) symbol should be ignored.

NOTE: To make things easier, you can jump to these lines in the file by typing Ctrl+W, typing PASS, and pressing Enter.


The meaning behind each of these parameters is explained in the file:
PASS_MAX_DAYS is the number of days that can elapse before a user has to change their password. This prevents users from eternally sticking to the same password, which can be a security risk if the password gets discovered.
PASS_MIN_DAYS is the number of days that have to elapse after a password change before a user can change it again. If you have password history settings configured (you really should), this prevents a user from repeatedly changing their password until an old one can be used again.
PASS_WARN_AGE is the number of days before PASS_MAX_DAYS that a user is urged to change their password.


PASS_MAX_DAYS is typically set to 30, 60, 90, or 120 days. In CyberPatriot, we stick to 30 days to be extra-secure.

PASS_MIN_DAYS can be set to 2 days.

PASS_WARN_AGE can be reasonably set to 7 days.
