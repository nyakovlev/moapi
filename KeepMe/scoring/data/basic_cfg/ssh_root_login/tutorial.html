Prohibiting Root Login Over SSH

Enforcing secure SSH server policies is a critical task when hardening the image, given that the system absolutely needs to be running an SSH server. If such is not the case, you should disregard this hardening task and simply PURGE openssh-server. (sudo apt-get purge openssh-server)
Note: if the README in the competition image states that sshd is a critical service, then openssh-server must be installed and secured.


To prohibit root login over ssh, edit the SSH server configuration file by typing sudo nano /etc/ssh/sshd_config


Go down to where it says PermitRootLogin (the default setting is prohibit-password) and change it so that it says PermitRootLogin no.


Save the file (Ctrl+O). That should do it!
