Removing Telnet

Referenced from page 142 of CIS Ubuntu Linux 16.04 Benchmark

Installing various packages can make your system vulnerable to any known or discovered security issues in the software. Hackers trying to gain access to your machine often can take advantage of such security holes in your system's software. For this reason, it is highly recommended to purge anything on your machine that is not required to be there.


Telnet is an insecure and unencrypted package that allows users to communicate to a system remotely. It has been replaced by the far more secure SSH protocol. This package will very likely need to be removed. (any telnet servers should certainly be removed as well; however, this tutorial only covers the telnet client)


To check if telnet client is installed on your system, you can type dpkg -l telnet.


If the name of the package is preceded by "ii", then it is installed. To remove it, simply type sudo apt-get purge telnet.


That should be it!
