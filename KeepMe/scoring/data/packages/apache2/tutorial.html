Removing Apache HTML server

Installing various packages can make your system vulnerable to any known or discovered security issues in the software. Hackers trying to gain access to your machine often can take advantage of such security holes in your system's software. For this reason, it is highly recommended to purge anything on your machine that is not required to be there.


Apache is an HTML server intended to provide users access to webpages and other resources. If the README specifically instructs you to keep Apache, steps should be taken to secure it, but if no mention is made, Apache should be removed entirely.


To check if Apache is installed on your system, you can type dpkg -l | grep apache. (the grep command is to find packages with names containing "apache")


If results come up, then Apache is likely installed. To remove Apache, purge the results that return from the above command. (For example: sudo apt-get purge apache2 apache2-bin apache2-data apache2-utils)


That should be it!
